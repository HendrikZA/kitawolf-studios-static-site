How it Works

Ultra-Performant, Zero Bloat

Most broken link checkers for WordPress are notorious for bloating database tables and hogging server CPU resources—often leading to hosting provider bans. Kitawolf Dead Links Checker & Fixer was designed from the ground up to solve this, keeping scans gentle, rate-limited, and hyper-focused on what it does best.

⚡

Performant Batch Scanning

Uses a client-driven AJAX batch scanner that processes posts in tiny chunks (with customizable rest intervals) to prevent CPU spikes and database lockups.

🎨

Premium React Dashboard

Restructured around a modern tabbed layout featuring global stats, interactive filtering, and real-time logs styled with custom typography.

🛠️

Direct Inline Repair

Edit broken URLs or strip link tags (unlinking) directly inside the explorer table without having to open individual post editors.

⏱️

Quiet Cron Automation

Schedule daily, weekly, or monthly scans that run quietly in the background. Administrators receive a single email notification only if broken links are discovered.

Smart Exclusions List

Help Center

Frequently Asked Questions

Will crawling slow down my website performance?

No. The crawler engine utilizes throttling parameters and chunk-based scheduling. It makes internal post loops and HTTP requests asynchronously using local cron execution, checking links in small sets to avoid page latency for regular users or exceeding hosting resource limits.

Can I edit links directly in Gutenberg/Page Builders?

Yes. The Inline Link Editor updates links directly inside your database records. It is fully compatible with the WordPress Block Editor (Gutenberg) as well as classic page builders (Divi, Elementor, WPBakery) because it targets the saved text contents safely.

Is there a limit on how many links it checks?

There is no arbitrary limit built into the plugin. The checker can audit thousands of links across massive databases, limited only by your hosting server's capacity to process cron routines.
